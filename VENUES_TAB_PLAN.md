# План: Вкладка "Площадки" + Анкета заказчика + Админка

> Дата: 2026-07-07
> Статус: Черновик плана (не для реализации)

---

## 1. Архитектура: Смета → Проект → Площадка

```
┌─────────────┐     INSERT/UPDATE     ┌─────────────┐
│  estimates  │ ─────────────────────→ │   projects  │
│  (мастер)   │   (триггер или хук)    │   (slave)   │
└─────────────┘                        └─────────────┘
        │                                    │
        │ FK estimate_id                     │ FK venue_id
        ↓                                    ↓
   ┌─────────┐                          ┌─────────────┐
   │estimate_items│                     │ venue_details│
   └─────────┘                          └─────────────┘
```

**Правило:** Смета пишет в проект. Проект ссылается на площадку. Площадка — независимый справочник.

### 2.2 Таблица `projects` (только доп.данные, без дублей)

```sql
create table projects (
  id uuid primary key default gen_random_uuid(),
  company_id uuid not null references companies(id) on delete cascade,
  estimate_id uuid not null unique references estimates(id) on delete cascade,
  venue_id uuid references venue_details(id) on delete set null,
  
  -- ТОЛЬКО данные, которых нет в смете:
  guest_count integer,
  expected_attendance integer,
  tech_rider text,
  stage_plan_url text,
  
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

create index idx_projects_estimate_id on projects(estimate_id);
create index idx_projects_company_id on projects(company_id);
create index idx_projects_venue_id on projects(venue_id);
```

### 2.3 Триггер автосоздания проекта из сметы

```sql
CREATE OR REPLACE FUNCTION sync_project_from_estimate()
RETURNS TRIGGER AS $$
BEGIN
  IF TG_OP = 'INSERT' THEN
    INSERT INTO projects (company_id, estimate_id, venue_id)
    VALUES (NEW.company_id, NEW.id, NULL)
    ON CONFLICT (estimate_id) DO NOTHING;
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER estimate_project_sync
AFTER INSERT ON estimates
FOR EACH ROW
EXECUTE FUNCTION sync_project_from_estimate();
```

### 4.3 Обработка ответа

```typescript
// Заказчик отправляет анкету → создаётся/обновляется venue_details
// + создаётся запись в venue_survey_responses (для истории)

interface VenueSurveyResponse {
  id: string;
  company_id: string;
  project_id: string;        // к какому проекту привязана
  venue_id: string | null;   // создана ли площадка
  respondent_name: string;     // кто заполнял
  respondent_email: string;
  respondent_phone: string;
  raw_data: Record<string, any>; // сырые ответы
  status: 'pending' | 'processed' | 'approved';
  created_at: string;
}
```

  'Глубина сцены (м)': number | null;
  'Высота подвеса (м)': number | null;
  'Тип подвеса': string;
  
  // Вместимость
  'Вместимость (чел.)': number | null;
  'Гримерки': number | null;
  
  // Логистика
  'Время заезда': string;
  'Парковка': string;

**Формат Excel:**
- Лист 1: "Анкета" — все ответы
- Лист 2: "Сводка" — ключевые параметры для быстрого просмотра
- Лист 3: "История" — предыдущие анкеты по этой площадке
